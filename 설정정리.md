## 관리자 패널 헤더 클릭 정렬 (2026-07-09, 커밋 6762d1c, prod READY)
- app/admin/page.tsx 단일 파일 수정. 헤더 16개 컬럼(#·수정버튼 제외) 클릭 시 **오름차순 ▲ → 내림차순 ▼ → 해제(원래 순서)** 3단계 순환. 미정렬 표시는 ↕.
- 구현: `SortKey` 타입 + `sortValue(row, key, volumes, clicks)` + `compareValues(a, b, dir)` (모듈 스코프). 정렬은 `filtered` 복사본 `visible`에만 적용 — `rows` 원본 불변.
- 규칙:
  - 값 없음(null)은 asc/desc 무관하게 **항상 맨 뒤**.
  - 문자열은 `localeCompare(..., 'ko')` (한글 가나다순).
  - 상태는 가나다순이 아니라 `STATUSES` 정의 순서(미노출→노출중→종료→진행X) 인덱스로 정렬.
  - 조회수(combined_views)는 null→0 취급이라 오름차순에서 '-'(0)이 앞에 온다.
- 파생값 정렬 지원: 검색량=`volumes[keyword].total`, 총 클릭수=`clicks[id]`, 총 노출일=`is_exposed` 카운트.
- 라이브 검증(Playwright 실클릭): 조회수 asc/desc/해제, 키워드 한글 정렬, 상태 정의순 모두 확인.

### 신규 컬럼 (amos_posts / median_posts 동일, 전부 콤마 구분 text)

### 조회수/클릭수 계산식 (변경)
```
조회수 = 카페 readCount 합(blog_url + past_urls 중 카페)
       + 이미지 조회수 합(image_host_url + past_image_host_urls)
       + views_base
클릭수 = totalVisits 합(hwaseon_url + past_hwaseon_urls)
```
- `views_base` = **라이브로 못 구하는 값만** 보관. ①블로그인데 이미지호스팅링크 없음 ②카페글 삭제됨 ③이미지 소스 리셋(라이브 < 엑셀)
- `clicks_base` 컬럼은 **만들지 않음** — 엑셀 클릭수 전 행이 단축코드 라이브 조회에 성공(실측). hwaseon-url totalVisits가 전체 누적이라 과거분 포함.
- `views_offset`은 더 이상 사용하지 않음 (전 행 0).
